# FBI: A Framework for Benchmark Integrity to Verify Malicious Contamination in LLMs

본 저장소는 LLM의 성능 검증 신뢰성을 높이기 위한 **'보안 벤치마크 생성 및 통계적 치팅 탐지 프레임워크(FBI)'** 원문과 실제 구축된 벤치마크 데이터셋을 아카이빙한 공간입니다. 


---

## 📂 저장소 파일 명세 

본 저장소에 포함된 데이터셋은 총 997개의 고품질 사지선다형 문항으로 구성되어 있으며, 설계 목적에 따라 두 가지 버전으로 관리됩니다. 

* `[_ICML_2026__FBI__A_Framework_for_Benchmark_Integrity_to_Verify_Malicious_Contamination_in_LLMs-3].pdf`: ICML 제출 당시의 논문 원문 (Full Paper) 

* `honeypot_dataset_simple.csv`: 비밀 키 기반 SHA-256 해시가 적용되어 선택지 순서가 의사난수로 변환된 기본 탐지용 데이터셋입니다. 모델의 정답 선택지 선호 편향 분포를 파악할 수 있는 고유 식별자(`original_perm_index`)가 포함되어 있습니다. 

* `honeypot_dataset_web.csv`: 지능형 우회 학습 공격에 대응하기 위해 계층 구조가 도입된 데이터셋입니다. 문항 간의 유기적 흐름과 조건부 탐지를 추적할 수 있도록 부모 문항 매핑 식별자(`parent_id`)가 추가 설계되어 있습니다.

## 📌 연구 개요
 
**연구 주제:** LLM 사전 학습(Pre-training) 또는 파인튜닝 과정에서 평가용 테스트 데이터가 고의적·비고의적으로 유출되어 모델 성능이 왜곡되는 문제를 감지하는 방법론 제시 

**진행 기간:** 2025.07 ~ 2026.01

**학회 제출 이력:** ICML(International Conference on Machine Learning) 투고 후, 현재는 연구 보완을 위해 Withdrawn(철회) 상태입니다.

## 📌 연구 배경 및 문제 정의

* **기존 공개 벤치마크(예: MMLU) 탐지의 한계 (정황 증거):** 어떤 모델이 특정 벤치마크의 원래 보기 순서를 일관되게 선호하더라도, 모델 개발사가 우연의 일치라고 주장할 경우 이를 완벽히 반박하기 어렵습니다. 기존의 n-gram이나 Perplexity 기반 탐지 기술은 확률적 추정에 의존하므로 악의적 오염을 입증할 법적·수학적 강제력이 부족합니다. 

* **비밀 키 기반 함정수사 벤치마크(FBI)의 핵심 이점 (부인 방지, Non-repudiation):** 본 프레임워크가 생성하는 벤치마크의 배열 순서는 외적인 사전식 순서나 논리적 흐름과 무관합니다. 오직 [질문($q$) + 비밀 키($K$)]의 조합으로 생성된 암호학적 의사난수(Pseudo-random)에 의해 결정됩니다. 따라서 모델이 이 특정 배열을 일관되게 선호한다면 우연이라는 변명은 통하지 않으며, 이는 워터마크와 같은 결정적 증거가 됩니다.

## 🛠️ 핵심 방법론 (Methodology) & 알고리즘 아키텍처
<img width="988" height="336" alt="image" src="https://github.com/user-attachments/assets/cfc779c8-14f3-4067-a8bd-3ca741227bb3" />


본 프레임워크는 객관식 문항(MCQA) 형태의 모든 벤치마크에 범용적으로 적용 가능하며, 암호론적 핑거프린팅을 통해 악의적인 학습자가 결과물만 보고는 탐지 패턴을 파악할 수 없도록 설계되었습니다. 

### 1. 암호론적 핑거프린팅 (Phase 1: Secure Benchmark Generation)
비밀 키($K$)와 질문($q$)을 결합하여 **SHA-256 해시 함수**를 통과시키고, 도출된 해시 값을 기반으로 문항마다 고유하고 예측 불가능한 선택지 순서(Permutation, $\pi^*$)를 확정 지어 표식을 남깁니다. 


### 2. 지능형 우회 공격 대응을 위한 계층적 강건성 확보 (Parent-Child Connection)

* **공격 시나리오:** 악의적인 학습자가 선택지 순서 의존성(Order Dependency)을 무력화하기 위해, 테스트셋의 정답 선택지 순서를 $n!$로 무작위 셔플(Shuffling)하여 학습하는 지능형 치팅을 시도할 수 있습니다. 
* **대응 메커니즘 (조건부 의존성 설계):** 이를 방어하기 위해 문항마다 부모 문항(Parent Question)을 지정하고, 부모 문항에 대한 조건부 의존성(Conditional Dependency)을 갖도록 설계했습니다.  함정의 발동 여부가 모델의 상위 맥락 이해도에 종속되므로, 단순히 선택지 순서만 기계적으로 섞어서 학습하는 가벼운 공격으로는 FBI의 표식을 무력화할 수 없습니다.



### 3. 통계적 검증 효율성(Efficiency) 극대화

* 기존 탐지 방식은 데이터 오염 여부를 확인하기 위해 Test Set 전체를 전수 조사해야 하므로 컴퓨팅 비용이 매우 컸습니다. 
* 반면, 본 계층 구조 프레임워크에서는 부모 문항의 정답 여부에 따라 함정이 발동하므로, **"모델의 실제 정답률"에 비례하여 검사해야 하는 데이터의 비율이 크게 감소**합니다. 결과적으로 탐지의 강건성(Robustness)을 획기적으로 높이면서도 통계적 검증의 효율성(Efficiency)까지 동시에 달성했습니다.

## 💻 주요 기여 및 수행 역할 (핵심 데이터 엔지니어링 및 파이프라인 구축)

본 연구의 방법론 설계 회의에 참여하여 아이디어를 구체화하였으며, 논문의 핵심 실험인 'Zero-base' 상태에서의 검증을 위한 대량의 사지선다형 벤치마크 데이터셋 구축 및 암호화 전처리 파이프라인을 전담하여 설계 및 구현했습니다. 

* **아이디어 회의 참여 및 방법론 구체화**
선행 연구 분석을 바탕으로, 벤치마크의 악의적 오염을 통계적으로 확정 짓기 위해 선택지 순서에 암호화 패턴을 심는 방안의 아이디어 미팅에 참여하고 프레임워크 구체화 과정에 기여했습니다. 

* **보안 벤치마크 생성을 위한 SHA-256 해시 알고리즘 적용 (Phase 1)**
프레임워크의 핵심인 암호론적 핑거프린팅을 구현하기 위해, **비밀 키($K$)와 질문($q$)을 결합하여 SHA-256 해시 값을 생성하고 이를 기반으로 선택지 순서(Permutation)를 결정하는 전처리 로직을 직접 코드로 구현하고 적용**했습니다. 

* **시드 데이터 확보 및 포맷 정의 (Seed Data Generation)**
Google NotebookQuiz를 활용하여 초기 퀴즈 데이터를 생성하고, 검증에 적합한 객관식 문항 포맷 및 정답 유형의 기준을 수립했습니다.


* **Few-shot Prompting 기반 품질 검증**
생성된 시드 데이터를 LLM(Gemini)에게 Few-shot 예시로 제공하여 출제 스타일을 유도했습니다.
모델이 절대 본 적 없는, 팀원의 자작 소설 원문을 기반으로 초·중·후반 핵심 에피소드를 모두 아우르는 고품질 검증 문항을 우선 제작하여 데이터 정밀도를 테스트했습니다. 

* **파이썬 기반 자동화 및 계층적 데이터 파이프라인 구축 (Context Chunking & Parent-Child Mapping)**
소설 원문 전체를 미시적 정보 유실 없이 커버하기 위해, 텍스트 데이터를 챕터와 페이지 구간별로 분할(Chunking)하는 파이썬 스크립트를 작성했습니다. 

* **부모 문항(대형 컨텍스트)과 자식 문항(세부 질문) 간의 계층적 연결 관계를 정의하고 조건부 의존성(Conditional Dependency)을 갖도록 매핑하는 전처리 파이프라인을 구현**했습니다. 
API 호출 시 50문항씩 배치 처리하는 프로세스를 통해, 소설 내 세부 정보까지 반영된 총 997문항의 대량 벤치마크 데이터셋(CSV)을 성공적으로 생성 및 정제했습니다.
